# TKXT — NFT Tickets

A minimal NFT-based event ticketing system running entirely on a local Hardhat blockchain with a vanilla HTML/JS frontend. No backend server required.

## Tech stack

- **Hardhat** — local EVM chain + compilation + deployment
- **Solidity 0.8.20** + **OpenZeppelin** — ERC-721 contract with `Ownable` and hash-based check-in
- **Vanilla HTML/JS** — single-file frontend, no build step
- **ethers.js v6** (CDN) — wallet + contract interactions
- **MetaMask** — wallet connection
- **qrcodejs** (CDN) — QR code generation

---

## Quickstart

### 1. Install dependencies

```bash
npm install
```

### 2. Start the local Hardhat chain

In **terminal 1**:

```bash
npm run chain
```

Hardhat will print 20 funded accounts. Copy account #0 — that is the **owner/deployer**. Import it into MetaMask (private key shown in terminal).

### 3. Add Hardhat network to MetaMask

| Field    | Value                  |
|----------|------------------------|
| RPC URL  | `http://127.0.0.1:8545` |
| Chain ID | `31337`                |
| Currency | `ETH`                  |

### 4. Compile + deploy

In **terminal 2**:

```bash
npm run dev
# (runs: hardhat compile && hardhat run scripts/deploy.js --network localhost)
```

This writes `frontend/contract.json` with the deployed address and ABI.

### 5. Open the frontend

Open `frontend/index.html` directly in your browser (double-click, or serve with any static server):

```bash
npx serve frontend
# then visit http://localhost:3000
```

## How it works

### Contract: `NFTTicket.sol`

| Function | Access | Description |
|---|---|---|
| `mint(to, eventName, checkInHash)` | Owner only | Mints ERC-721 ticket with pre-committed hash |
| `checkIn(tokenId, providedHash)` | Owner only | Validates hash and marks ticket used on-chain |
| `getTicket(tokenId)` | Public | Returns owner, event, hash, used status |
| `totalMinted()` | Public | Returns count of all minted tokens |

### Hash-based check-in

When minting, the frontend:
1. Generates a random 32-byte `secret`
2. Computes `checkInHash = keccak256(abi.encodePacked(secret, tokenId))`
3. Sends `checkInHash` on-chain at mint time
4. Embeds `{ tokenId, checkInHash }` in the QR code shown to the ticket holder

At check-in, the scanner:
1. Scans the QR (or pastes the hash manually)
2. The contract compares `providedHash == ticketHash[tokenId]`
3. If valid and unused → marks the ticket used and emits `TicketCheckedIn`

The raw `tokenId` is never enough on its own — you need the hash too.

### Owner-only protection

- `mint` and `checkIn` both use OpenZeppelin's `onlyOwner` modifier
- The frontend checks `contract.owner()` at wallet connect time
- The **Admin** tab and check-in write path are gated to the owner wallet in the UI

## Notes

- Restarting `npm run chain` resets all state. Re-run `npm run dev` to redeploy.
- The check-in hash is stored in the browser's `localStorage` keyed by token ID so QR codes survive page reloads — clear storage to reset.
- For production, replace `localhost` with a real network in `hardhat.config.js` and load secrets from a `.env` file (never commit private keys).
